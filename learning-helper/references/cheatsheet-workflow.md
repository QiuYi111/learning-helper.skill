# Cheat Sheet 工作流

场景 A 的完整五阶段管线。主代理按此执行。

---

## 阶段 1：范围确认

开始前务必和用户明确：

- **课程名**和**考试类型**（期中 / 期末 / 小测）
- **目标页数**（例：期中 3-4 页，期末 5-6 页）
- **Source 覆盖范围**：Knowledge/Examples/Expansions 各有哪些文件？是否全部纳入，还是只覆盖特定章节？
- **重点章节**：用户心里有没有"这次考试重点会在哪几章"的判断？记录但不完全依赖（避免被主观判断误导）
- **已知的排版偏好偏离**：理论上排版是固定的，但如果用户这次想破例（比如单栏、更大字号），现在问清楚

不要跳过这一步。没有这些信息，后续蒸馏会没有方向。

---

## 阶段 2：并行蒸馏 Source

对 `Source/Knowledge/` 和 `Source/Examples/` 下的每份材料，**并行启动 subagent** 提取内容，各自产出一份 md 保存到 `Artifacts/distillation/` 下。

### 每个 subagent 的任务模板

给每个 subagent 的 prompt 应包含：

```
你是一个知识蒸馏子代理。你的任务是从给定材料中提取**所有**值得收入 cheat sheet 的内容，写成一份结构化的 md 文件。

材料路径：{file_path}
材料类型：{Knowledge | Examples | Expansions}
输出路径：Artifacts/distillation/{filename}.md

要求：
1. 完整性优先。宁可多提取后面再删，不要漏。课件里讲过的每个定义、性质、公式、定理、例题都要记录。
2. 保留原始结构（章节、编号），方便后续交叉引用。
3. 数学公式用 LaTeX 记号（后续由主代理转 OMML）。
4. 如果是 Examples 类材料（作业/习题课/样题），额外标注每道题属于什么题型、用了什么方法。
5. 不要做判断性筛选——你的任务是"全"，筛选由主代理在阶段 4 完成。

输出格式：
# {材料名}

## 章节 1：{标题}

### 定义 / 性质 / 公式
- ...

### 例题 / 题型（仅 Examples 类）
- 题目描述
- 题型标签：{e.g. 非齐次方程 + 齐次边界}
- 解法骨架：...

（依此类推）
```

### 并行策略

- 每个文件一个 subagent，同时启动
- 不要让一个 subagent 处理多个文件（容易混淆来源）
- Expansions 文件也单独起 subagent，但标签为 "Expansions" 以便主代理区分权重

### 等待和汇总

所有 subagent 完成后，主代理扫一遍 `Artifacts/distillation/` 目录，确认每份 Source 都有对应的 distillation md。缺失的重启。

---

## 阶段 3：二级产出（题型 + 二级结论）

两个独立 subagent 任务，**并行**执行。基于阶段 2 的 distillation md 进行二次加工。

### 任务 3a：题型模板挖掘

```
你是题型挖掘子代理。基于已有的 distillation md 文件（特别是 Examples 类），归纳出**一类题一个模板**。

输入：Artifacts/distillation/ 下所有 md 文件
输出：Artifacts/templates.md

要求：
1. 先扫一遍所有 Examples，归纳题型。同类题合并到一个模板里，不要一道题一个。
   - 例如：PDE 里"非齐次方程 + 齐次 Dirichlet 边界"和"非齐次方程 + 齐次 Neumann 边界"可以是同一个模板，只是边界条件有分支。
2. 每个模板包含：
   - 题型标签（简洁、可识别）
   - 识别条件（看到什么特征就用这个模板）
   - 解题骨架（步骤编号 ①②③④⑤）
   - 典型陷阱（这类题常翻车的地方）
   - 一道代表性例题（最好从 Examples 里选，注明来源）
3. 如果一道例题能套两个模板，记录两个来源（"参见模板 X 和 Y"）。
4. 对同类题中**难度差异较大**的变体，在同一模板下列出难度层级。

格式：
# 题型模板

## 模板 1：{题型名}

**识别**：...
**骨架**：
  ① ...
  ② ...
  ③ ...
**陷阱**：...
**例题**：{出处} - {简述}
```

### 任务 3b：二级结论挖掘

```
你是二级结论挖掘子代理。基于 distillation md 文件，挖掘课件里没有显式写出、但由知识点/例题衍生出来的快速解题结论。

输入：Artifacts/distillation/ 下所有 md 文件
输出：Artifacts/secondary-conclusions.md

要求：
1. 只收录真正有用的、能显著加速解题的结论。不收录"显而易见"的或"教材已经显式写了"的。
2. 每条结论注明：
   - 结论本身（精确表述）
   - 推导来源或证明思路（一行）
   - 使用场景（什么题型能用）
3. 如果某个 Expansions 文件里有用户自己悟出的结论，优先收录。
4. 如果不确定某结论是否正确，标注"待验证"而不是直接收录。
```

两个任务可以并行跑。

---

## 阶段 4：主代理综合

主代理读取：

- `Artifacts/distillation/` 下所有 md
- `Artifacts/templates.md`
- `Artifacts/secondary-conclusions.md`

综合成一份 `Artifacts/cheatsheet-draft.md`。

### 组织原则

按 SKILL.md 里规定的内容优先级组织：

1. 基本定义 / 性质 / 变换对表
2. 核心公式和恒等式
3. 方法论 / 流程
4. 题型模板
5. 二级结论
6. 易错 / 陷阱
7. 速查附录

### 编号策略

从 1 开始跨章节连续编号。每条内容占一个编号。草稿阶段不用担心编号对齐，后续排版时会自动重排。

### 页数预算

粗略估算：A4 双栏 0.8cm 边距 + 10pt 正文，每页约容纳 20-25 个条目（依公式密度浮动）。目标 75 条对应约 3-4 页。

**如果草稿条目数明显超出目标页数对应的容量**，主代理先自查，按 SKILL.md 的"优先砍"规则裁剪后再进入阶段 5。

---

## 阶段 5：宽松覆盖审核 + docx 输出

### 审核（放宽版）

启动一个审核 subagent：

```
你是 cheat sheet 审核子代理。

输入：
- Source/Knowledge/ 和 Source/Examples/ 下的所有原始材料（或它们的 distillation md）
- Artifacts/cheatsheet-draft.md

任务：
通读两边，**汇报明显缺失**。"明显缺失"指：Source 里讲过、有一定分量、但草稿里根本没出现的内容。不要逐条对照——那会导致你陷入细节。

输出：Artifacts/coverage-review.md，列出：
- 明显缺失项（如果有）：{内容} + {出处}
- 可疑弱覆盖项（比如只提了一句但 Source 里是重点）：{内容} + {建议}
- 整体评价：覆盖良好 / 有明显漏洞 / 需大幅补充

**不要阻塞**——这只是给主代理的建议清单，主代理自行决定是否采纳。
```

### 主代理处理审核反馈

- 如果审核说"覆盖良好"：直接进入排版
- 如果有"明显缺失项"：主代理判断是否重要，重要就补进草稿，不重要就忽略
- 如果审核说"有明显漏洞"或"需大幅补充"：回阶段 4 重写相关章节

### 最终排版输出 docx

使用 `processing/docx` skill 生成 docx。

**关键排版要点**（再次强调）：

- A4 双栏，0.8cm 边距
- 条目连续编号，Heading 1/2/Body Text 三级样式
- **所有数学符号用 OMML 公式**，不要纯文本近似
- 标题字号 12-14pt，正文 10-11pt
- 章节之间留少量空白但不要太大，保持密度

输出路径：`Artifacts/cheatsheet.docx`（或用户指定的名字）。

输出后用 `present_files` 呈现给用户。

---

## 失败处理

- **subagent 超时或报错**：重试一次，再失败就跳过该文件并在最终产物里标注"未成功蒸馏 {file}"
- **找不到 docx skill**：提示用户先用 `find-skill` 安装，阻塞
- **Source 目录为空**：停止，问用户材料在哪
- **用户提供的材料格式不支持**（比如扫描 PDF 图片）：提示用户，建议用 OCR 工具或手动转录后放进 Source

---

## 增量更新（不主动支持，但用户要求时的应对）

本 skill **不主动维护增量更新机制**。如果用户要求"基于现有 cheat sheet 加一章"，按以下简化流程：

1. 读 `Artifacts/cheatsheet.docx`（或对应 md）
2. 对新增材料走阶段 2-3
3. 合并到现有草稿
4. 跳过审核或简化审核
5. 出新 docx

不要把"增量更新"当作长期维护任务——每次考试重新生成一次更稳妥。
